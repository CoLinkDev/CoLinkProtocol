# Cloud WebSocket

Real-time communication channel between device and cloud server.

## Connection

```
wss://server/ws/v1?ticket=<ticket>
```

Obtain a `ticket` via `POST /api/v1/ws/ticket` (see `ticket.md`). The ticket is single-use and expires after 30 seconds.

## Message Envelope

All messages use a unified JSON format:

```json
{
  "id": "a1b2c3d4-e5f6-7890-abcd-ef0123456789",
  "type": "message.type",
  "from": "deviceId",
  "to": "deviceId | null",
  "payload": {},
  "timestamp": 1716451200000
}
```

| Field     | Type        | Description                                 |
|-----------|-------------|---------------------------------------------|
| id        | string      | Message UUID (for dedup, ACK, correlation)  |
| type      | string      | Message type identifier                     |
| from      | string/null | Sender device ID (server fills for inbound) |
| to        | string/null | Target device ID (null = broadcast)         |
| payload   | object      | Message-specific data                       |
| timestamp | number      | Unix timestamp in milliseconds              |

## Lifecycle

1. Client obtains ticket via `POST /api/v1/ws/ticket`
2. Client connects with `ticket` in query string
3. Server validates ticket → connection established → device marked online
4. Server broadcasts `device.online` to same-account devices
5. Client sends `ping` every 30s to keep alive
6. On disconnect → server broadcasts `device.offline`

## Server → Device Messages

### device.online

```json
{
  "id": "b2c3d4e5-f6a7-8901-bcde-f01234567890",
  "type": "device.online",
  "from": "660e8400-...",
  "to": null,
  "payload": {
    "name": "My Phone",
    "type": "android"
  },
  "timestamp": 1716451200000
}
```

### device.offline

```json
{
  "id": "c3d4e5f6-a7b8-9012-cdef-012345678901",
  "type": "device.offline",
  "from": "660e8400-...",
  "to": null,
  "payload": {},
  "timestamp": 1716451200000
}
```

### relay (inbound)

Message relayed from another device:

```json
{
  "id": "d4e5f6a7-b8c9-0123-defa-123456789012",
  "type": "relay",
  "from": "660e8400-...",
  "to": "770e8400-...",
  "payload": {
    "type": "clipboard.v1.sync",
    "payload": { "contentType": "text/plain", "content": "Hello" }
  },
  "timestamp": 1716451200000
}
```

Inner `payload` is a business protocol message.

### pong

```json
{ "id": "e5f6a7b8-c9d0-1234-efab-234567890123", "type": "pong", "timestamp": 1716451200000 }
```

## Device → Server Messages

### ping

```json
{ "id": "f6a7b8c9-d0e1-2345-fabc-345678901234", "type": "ping" }
```

### relay (outbound)

Request server to forward a message to another device:

```json
{
  "id": "a7b8c9d0-e1f2-3456-abcd-456789012345",
  "type": "relay",
  "to": "770e8400-...",
  "payload": {
    "type": "message.v1.text",
    "payload": { "messageId": "88a3c4d5-e6f7-8901-abcd-ef0123456789", "text": "hi" }
  }
}
```

Server fills `from` and delivers. If target is offline, message is dropped (no offline queue in MVP).
