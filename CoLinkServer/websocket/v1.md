# Cloud WebSocket

Real-time communication channel between device and cloud server.

## Connection

```
wss://server/ws/v1?deviceId=<deviceId>&deviceSecret=<deviceSecret>
```

## Message Envelope

All messages use a unified JSON format:

```json
{
  "type": "message.type",
  "from": "deviceId",
  "to": "deviceId | null",
  "payload": {},
  "timestamp": 1716451200000
}
```

| Field     | Type        | Description                                 |
|-----------|-------------|---------------------------------------------|
| type      | string      | Message type identifier                     |
| from      | string/null | Sender device ID (server fills for inbound) |
| to        | string/null | Target device ID (null = broadcast)         |
| payload   | object      | Message-specific data                       |
| timestamp | number      | Unix timestamp in milliseconds              |

## Lifecycle

1. Client connects with `deviceId` + `deviceSecret`
2. Server validates → connection established → device marked online
3. Server broadcasts `device.online` to same-account devices
4. Client sends `ping` every 30s to keep alive
5. On disconnect → server broadcasts `device.offline`

## Server → Device Messages

### device.online

```json
{
  "type": "device.online",
  "from": "660e8400-...",
  "payload": {
    "name": "My Phone",
    "type": "android",
    "localIp": "192.168.1.5",
    "localPort": 27777
  }
}
```

### device.offline

```json
{
  "type": "device.offline",
  "from": "660e8400-...",
  "payload": {}
}
```

### relay (inbound)

Message relayed from another device:

```json
{
  "type": "relay",
  "from": "660e8400-...",
  "to": "770e8400-...",
  "payload": {
    "type": "clipboard.v1.sync",
    "payload": { "contentType": "text/plain", "content": "Hello" }
  }
}
```

Inner `payload` is a business protocol message.

### pong

```json
{ "type": "pong" }
```

---

## Device → Server Messages

### ping

```json
{ "type": "ping" }
```

### relay (outbound)

Request server to forward a message to another device:

```json
{
  "type": "relay",
  "to": "770e8400-...",
  "payload": {
    "type": "message.v1.text",
    "payload": { "messageId": "uuid", "text": "hi" }
  }
}
```

Server fills `from` and delivers. If target is offline, message is dropped (no offline queue in MVP).

### announce

Report local network address for LAN discovery:

```json
{
  "type": "announce",
  "payload": {
    "localIp": "192.168.1.3",
    "localPort": 27777
  }
}
```

Server includes this info in `device.online` broadcasts.
